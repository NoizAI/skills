# Contributing

欢迎贡献新的技能或改进现有技能。  
Contributions for new skills and improvements are welcome.

## 提交前检查 / Pre-Submission Checklist

- 新增技能放在 `skills/<skill-name>/SKILL.md` / Put new skills under `skills/<skill-name>/SKILL.md`
- 技能目录命名使用小写短横线（kebab-case） / Use kebab-case for skill directory names
- `SKILL.md` 必须包含 / `SKILL.md` must include:
  - 技能简介 / Skill overview
  - 触发词（triggers） / Triggers
  - 能力说明（capabilities） / Capabilities
  - 执行步骤（workflow） / Workflow
  - 限制和注意事项（limitations） / Limitations and notes

## 建议流程 / Suggested Flow

1. Fork 或新建分支 / Fork or create a branch
2. 新增/修改技能文档 / Add or update skill docs
3. 自查格式与可读性 / Self-check formatting and readability
4. 提交 PR，说明变更目的和使用场景 / Open a PR with purpose and use cases

## 评审重点 / Review Focus

- 触发词是否覆盖真实场景 / Whether triggers map to real usage
- 执行步骤是否明确且可操作 / Whether workflow is clear and actionable
- 是否避免过度承诺（写清边界） / Whether boundaries are explicit and realistic
